# ask-codex — Claude Code Skill

A Claude Code Skill that bridges [Codex CLI](https://github.com/openai/codex) (GPT-5) into your Claude Code workflow, giving you an independent second opinion from a different AI model.

## What It Does

- **Review Plans** — Have Codex audit implementation plans generated by Claude
- **Review Test Strategies** — Get independent feedback on test coverage and design
- **Create New Plans** — Ask Codex to draft plans from scratch
- **Code Review** — Run Codex's code review on uncommitted changes or branch diffs
- **General Tasks** — Use Codex as an independent agent for any task

## Prerequisites

1. **Codex CLI** installed (`npm install -g @openai/codex`)
2. **Codex authenticated** (`codex login` or configure `~/.codex/config.toml`)
3. **WSL** (Windows only) — The skill runs Codex via WSL due to a known Windows native binary crash ([#12962](https://github.com/openai/codex/issues/12962)). On Linux/macOS, you may need to adapt the WSL wrapper calls.

## Installation

Copy the skill into your Claude Code skills directory:

```bash
# Option 1: Clone and copy
git clone https://github.com/Simpson-wu/codex-skill.git
cp -r codex-skill ~/.claude/skills/ask-codex

# Option 2: Direct install (if Claude Code supports it)
# Add to your Claude Code settings under skills
```

Make sure the skill directory contains at minimum:
```
ask-codex/
├── SKILL.md              # Core skill definition (required)
├── CLAUDE.md             # Project context for development
├── prompting-guide.md    # How to craft effective Codex prompts
├── templates/            # Ready-to-use prompt templates
│   ├── review-plan.md
│   ├── review-test.md
│   └── code-review.md
└── docs/                 # Design docs & review history
    └── codex-review-v1.md
```

## Usage

In Claude Code, invoke the skill with:

```
/ask-codex Review this plan: [your plan content]
/ask-codex What are the security implications of this approach?
/ask-codex Review the uncommitted changes in this repo
```

Or let Claude auto-trigger it when the context matches (e.g., when asking for a second opinion).

## How It Works

1. **Environment Detection** — Discovers WSL distro, Codex binary path, and config
2. **Intent Analysis** — Maps your request to one of 4 execution modes:
   - **Mode A**: Short text via stdin (read-only sandbox)
   - **Mode B**: Long text via temp file + stdin (read-only sandbox)
   - **Mode C**: Code review (`codex review`)
   - **Mode D**: File operations (`--full-auto`, requires confirmation)
3. **Execution** — Runs Codex CLI through WSL with proper path translation
4. **Output Extraction** — Strips banners/thinking, returns clean response

## Security

- Default sandbox is **read-only** — Codex cannot modify your files
- File-write mode (Mode D) requires explicit user confirmation
- All prompts are passed via stdin to prevent shell injection
- The `--ephemeral` flag ensures no session history leakage

## Customization

Edit `~/.codex/config.toml` to configure:
- Model provider and endpoint
- Default model (e.g., `gpt-5`)
- Reasoning effort level
